# 툴셋 설치 간편가이드

> 레거시 JSP "옷입히기" 작업용 하네스. 작업자 PC마다 1회 셋업.
> 목표: 아래 다 깔고 `검증` 섹션 통과하면 작업 시작 가능.

---

## 0. 전제 (이미 있어야 하는 것)

| 항목 | 용도 | 확인 |
|------|------|------|
| Claude Code | 하네스 본체 | `claude --version` |
| git | 형상/브랜치 | `git --version` |
| JDK + 톰캣(또는 프로젝트 빌드툴) | **레거시 앱 로컬 구동** | 프로젝트 README 기준 |
| 대상 소스 + 개발계 접근(VPN) | 작업 대상 | 프로젝트별 |

> ⚠️ **로컬에서 앱이 실제로 뜨는 것**이 모든 검증의 전제다. 이거 안 되면 `/qa`·`/browse` 불가 = 작업 불가.

---

## 1. 플러그인 설치 (2개)

작업에 실제로 쓰는 건 딱 두 묶음이다. (그 외는 이 작업엔 불필요)

| 플러그인 | 제공 스킬 | 역할 |
|----------|-----------|------|
| **gstack** | `/qa`, `/browse`, `/review` | UI 검증 · 시각비교 · diff 검토 |
| **context7** (MCP) | 라이브러리 문서 조회 | JSP/JSTL/EL 정확 문법 실시간 확인 |

설치 (본인 마켓플레이스/플러그인 설치 방식대로):

```text
# Claude Code 안에서
/plugin            # 마켓플레이스에서 gstack, context7 설치
```

> 설치 방식은 환경마다 다름(글로벌 vs vendored). **이미 세팅된 본인 환경과 동일하게** 깔면 됨.
> context7가 MCP 단독이면 `.mcp.json`에 등록되어 있는지 확인.

---

## 2. 검증 (설치 끝났는지 확인)

Claude Code에서 아래가 다 보이면 OK:

```text
□ /qa        호출 가능
□ /browse    호출 가능
□ /review    호출 가능
□ context7   MCP 연결됨 (라이브러리 문서 질의 응답)
□ git 브랜치 생성/커밋 됨
□ 레거시 앱이 로컬 톰캣에 뜨고 브라우저로 페이지 열림
```

전부 체크되면 → **작업 가이드(WORKFLOW.md)로**.
